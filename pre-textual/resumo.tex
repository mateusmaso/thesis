\textoResumo{
  O texto do resumo deve ser digitado,
  em um único bloco,
  sem espaço de parágrafo.
  O resumo deve ser significativo,
  composto de uma sequência de frases concisas,
  afirmativas e não de uma enumeração de tópicos.
  Não deve conter citações.
  Deve usar o verbo na voz passiva.
  Abaixo do resumo,
  deve-se informar as palavras-chave
  (palavras ou expressões significativas retiradas do texto)
  ou, termos retirados de thesaurus da área.
  
  Falar sobre clientes que buscam estruturas de dados. Falar que hoje estes dependem de conhecer o acesso ao invés de pensar em termos de estrutura. 
  
  O que eu preciso? e não como vou fazer para acessar.
  
  Pensar em estrutura de dados, e não fluxo de dados.
  
  REST => fluxo de dados (que URI preciso acessar pra retornar a estrutura X)
  SOAP/RPC => fluxo de dados (que método preciso acessar pra retornar estrutura X)
}
\palavrasChave{
  GraphQL,
  REST,
  JSON Schema.
}
